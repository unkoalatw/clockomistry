# 🕒 Clockomistry 專案開發與優化總結 (2026-03-02)

這份文件總結了我們在這段對話中對 Clockomistry 進行的所有升級與重構，方便你在下一個對話接續開發。

## 🚀 已完成的核心優化與新功能

### 1. 介面與排版體驗升級
- **全域巨大時鐘顯示**：不管是否開啟「專注模式 (Zen Mode)」，時鐘、計時器、番茄鐘等核心時間顯示元件，現在都會**永遠保持最大且最具視覺衝擊力的大小**（這也符合了最初附圖的需求）。取消了過去縮小狀態的設定。
- **無縫日曆事件新增**：徹底重寫了月曆的操作邏輯。現在只要在主視圖點擊「任何一天」的數字，系統會自動跳轉至「事件與倒數」列表，並自動將該日期填入新增事件的輸入框。
- **專注模式過渡流暢化**：補齊了所有切換到專注模式時的 CSS `opacity` 動畫效果（包含頂部分頁列、番茄鐘的控制子選單等），並使用 `cubic-bezier` 縮放曲線，讓進入與退出專注模式完全不再有生硬的閃爍感。
- **智慧裝置提示**：專注模式結束提示會自動判斷環境，桌面版會寫「移動滑鼠結束」，而在觸控裝置（手機 / APK）上則會改為「點擊螢幕結束」。
- **版面高度適配修正**：大幅解決了先前各個模式（含多重計時器）下需要不斷上下捲動的問題，現在畫面會自適應停留在單一螢幕畫面內，不產生多餘捲軸。

### 2. 結構重構與維護性升級 (非常重要)
- **程式碼拆分 (Code Splitting)**：原本長達 2000 多行的單一 `script.js` 對於維護來說負擔過重。**現在已經將其成功拆卸為模組化文件**，全部放置於 `src/` 資料夾內！
- 全新的專案結構如下：
  - `src/00-imports.js`：所有的 React 與套件引入
  - `src/01-utils.js`：indexedDB、日期計算與其他輔助函式
  - `src/02-i18n.js`：18國語言與字面量字典
  - `src/03-constants.js`：各式主題、變數、預設字體與世界時區常數
  - `src/04-components.js`：`ClockDisplay`, `ProgressRing`, `WeatherWidget`, `NavigationBar` 等獨立的 UI 元件
  - `src/05-hooks.js`：`useAudio`, `useLocalStorage` 以及生命週期管理
  - `src/06-app.js`：最主要的 `<App />` 主體容器與邏輯
  - `src/07-index.js`：React DOM Render 入口建立
- **建置流程 (Build Process)**：Babel 已經設定為編譯整個 `src/` 資料夾。所有的獨立小文件會被自動打包成最終的 `script.dist.js`。

## ⚙️ 接下來的開發建議 (給下一個對話的提示)
1. 如果要修改**主程式邏輯與主要渲染**：請查閱並修改 `src/06-app.js`。
2. 如果要修改**底下導覽列或時鐘主字體**：請前往 `src/04-components.js` 修改相關 UI 元件。
3. 如果遇到**狀態或是播放音效**問題：可檢視 `src/05-hooks.js`。
4. **建置指令**：`npm run build`。

## 🛠️ 部署與同步 SOP (重要指令)

為確保專案同步與發布，請遵循以下步驟：

### 1. 重新建置專案 (Babel & Tailwind)
```bash
npm run build
```

### 2. 推送至 GitHub 備份
```bash
git add .
git commit -m "你的更新說明"
git push
```

### 3. 同步至 Android APK 專案
需在 `clock-apk` 目錄下執行同步腳本（建議在 PowerShell 執行）：
```powershell
cd ..\clock-apk
pwsh .\sync.ps1
cd ..\clock
```
